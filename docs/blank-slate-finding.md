# Blank-slate read: what the persona buys, and how to test for contamination

A de-primed advisor persona ("Claire") buys a **stance**, not added analytical quality: instructing the model to "reason only from the brief, you know nothing here" makes the read more naive by design — it reasons the same conclusions from first principles but suppresses the named frameworks an informed read would cite. The blank-stranger stance is the deliverable; choose it deliberately, knowing it trades expertise for a fresh, unanchored angle.

A custom subagent inherits whatever standing instructions the host has loaded, so it is fair to ask whether that context covertly bends the read. The lesson on testing: **probe contamination distributionally, never by reading one output for a "tell."** A single run can diverge by chance and a clean-looking output is not proof of clean reasoning — run many reads per condition and check whether the conditions are distinguishable at all. In practice the de-priming discipline (strip your answer, neutralise the framing, de-jargon) is the load-bearing isolation; inherited standing context is a far weaker contamination vector than anchoring intuition suggests.

## A stronger contamination vector: harness-injected environment state (2026-06-20)

The "inherited standing context is weak" conclusion above has an exception worth its own line. The harness injects an **environment block into every subagent** — working directory, current git branch, recent commit messages — the same preamble the main agent gets. A context-starved critic has no file or git tools, so it cannot *fetch* any of this; but it can passively *read* the injected block, and it will reason from it. The personas said "no files, no tools, no knowledge beyond the brief" — which covers *tools* but not *ambient signal the model can simply see*. (Issue `2026-06-20_0205`: a `/claire:blank` read opened by quoting a git branch, `feature/graph-purpose-primer`, that was never in its brief, and built a whole "this is builder-momentum, not a real need" thread on it.)

Two empirical refinements from reproduction:

- **It is conditional on semantic adjacency, not mere presence.** A blank-slate read dispatched from a repo on an *unrelated* branch (`main`, off-topic commits) ignored the ambient block cleanly across repeated samples — the model treats unrelated env state as noise. The leak fires when the ambient state is *adjacent* to the question, and the **git branch name is the salient trigger**: short, declarative, and read as a statement of the asker's intent in a way a longer commit log is not. This is a common real condition — feature branches are routinely named after the feature being decided.
- **It reproduces distributionally.** With a telling env block placed above an otherwise-neutral brief, the pre-fix `blank-slate-advisor` persona reasoned from the branch name **3/3**; pinned as `tests/evals/fixtures/blind_read_ambient_env_leak.json`. The fix is one clause added to every context-starved critic: ignore ambient signal (cwd / repo / branch / commits / session metadata), treat a name that resembles the question as coincidence, reason only from the brief. It only *subtracts* signal, so it stays inside the persona's permitted no-prior-knowledge declaration rather than adding priors.

General lesson for any starved-critic design: "you have no tools" is not the same instruction as "ignore what your environment shows you." A subagent sees more than its brief; the persona has to name the ambient channel explicitly and tell the model to disregard it.
